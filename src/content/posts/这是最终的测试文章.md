check_mount1.sh

```
#!/bin/sh

# 定义挂载点和 NFS 服务器地址
MOUNT_POINT="/userdata/nfs-ha"
NFS_SERVER="192.168.1.222:/userdata/nfs-ha"

# 定义检查间隔（秒）
step=0.5 # 减少到半秒

# 检查文件系统是否挂载
check_mount() {
  sudo mountpoint -q $MOUNT_POINT
}

# 重新挂载 NFS 文件系统
remount_nfs() {
  echo "NFS mount not found, attempting to remount..."
  sudo umount $MOUNT_POINT 2>/dev/null
  sudo mount -t nfs $NFS_SERVER $MOUNT_POINT
}

# 主循环
while true; do
  if ! check_mount; then
    remount_nfs
  fi
  sleep $step
done

```

```
            command:
            - test
            - -d

          initialDelaySeconds: 10
          periodSeconds: 1
          failureThreshold: 2
      volumes:
      - name: nfs-storage
        hostPath:
          path: "/userdata/nfs-ha"

```

`livenessProbe` 定义了一个基于命令的存活探针，用于检查容器是否存活。这个探针通过执行一个命令来检查 `/mnt` 目录是否存在。以下是您配置的详细解释：

- `exec`: 指定使用命令执行的方式来检查容器的存活状态。
- `command`: 指定要执行的命令，这里是 `test -d /userdata/nfs-ha。这个命令检查 `/userdata/nfs-ha` 目录是否存在。如果 `/userdata/nfs-ha` 存在，命令返回 0，表示容器存活；如果 `/userdata/nfs-ha` 不存在，命令返回非 0 值，表示容器不存活。
- `initialDelaySeconds`: 指定在容器启动后等待多少秒开始执行第一次存活检查。这里是 10 秒。
- `periodSeconds`: 指定存活检查的时间间隔。这里是每隔 1 秒执行一次检查。
- `failureThreshold`: 指定当连续多少次检查失败后，Kubernetes 将认为容器不存活。这里是连续 2 次检查失败后，容器将被视为不存活。

这个配置意味着，容器启动后 10 秒开始，每隔 1 秒 Kubernetes 将执行 `test -d /userdata/nfs-ha` 命令来检查 `/userdata/nfs-ha` 目录是否存在。如果连续 2 次检查发现 `/userdata/nfs-ha 不存在，Kubernetes 将认为容器不存活，并可能重启容器。
